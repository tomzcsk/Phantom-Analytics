# คู่มือออกแบบ UX/UI — Phantom Analytics

**เวอร์ชัน:** 1.1.0
**อัพเดตล่าสุด:** 2026-03-26

---

## ปรัชญาการออกแบบ

> น้อยแต่มาก ทุก pixel ต้องมีเหตุผล

Dashboard ของ Phantom Analytics ยึดหลัก 3 ข้อ:

1. **ชัดเจนก่อนครบถ้วน** — แสดงข้อมูลสำคัญก่อน ให้ผู้ใช้เจาะลึกได้เอง
2. **หนาแน่นแต่ไม่รก** — จัดข้อมูลอย่างมีประสิทธิภาพ ไม่มีพื้นที่เปล่า ไม่มีของตกแต่ง
3. **นำไปใช้งานได้** — ทุกกราฟต้องตอบคำถามได้ และแนะนำสิ่งที่ควรทำต่อ

---

## ภาษาที่ใช้

- UI ทั้งหมดเป็น **ภาษาไทย**
- ศัพท์เฉพาะใช้ภาษาอังกฤษ (เช่น Tracker, API, SSE, Funnel)
- ใช้คำที่เข้าใจง่ายแทนศัพท์เทคนิค:
  - "อัตราตีกลับ" → **"เข้าแล้วออก"**
  - "เซสชันเฉลี่ย" → **"อยู่นานเฉลี่ย"**
  - "pageview" → **"เข้าชม"**
  - "scroll" → **"เลื่อน"**
  - "click" → **"คลิก"**
  - "session_end" → **"ออก"**
- เวลาแสดงเป็นสัมพัทธ์ เช่น "3 นาทีที่แล้ว", "ตอนนี้"

## รายละเอียด Component

### การ์ด KPI (5 การ์ดในหน้าภาพรวม)

```
┌────────────────────────────┐
│  🟢 ออนไลน์ตอนนี้           │  ← ไฟเขียวกะพริบ
│                            │
│  47                        │  ← Display font, สีเขียว
│                            │
│  ใน 5 นาทีที่ผ่านมา         │  ← 12px, สีจาง
└────────────────────────────┘

┌────────────────────────────┐
│  เข้าแล้วออก            ↘   │  ← ลูกศรขึ้น/ลง (เขียว/แดง)
│                            │
│  32.5% (8)                 │  ← เปอร์เซ็นต์ + จำนวนคนในวงเล็บ
│                            │
│  -5.2% เทียบช่วงก่อนหน้า    │  ← 12px, สีตามทิศทาง
└────────────────────────────┘
```

**5 การ์ดในหน้าภาพรวม:**
1. ออนไลน์ตอนนี้ (realtime, สีเขียว)
2. การเข้าชม
3. ผู้เข้าชมไม่ซ้ำ
4. อยู่นานเฉลี่ย
5. เข้าแล้วออก (แสดงจำนวนคน)

### กราฟ Trend หลัก
- ประเภท: Line chart + area fill (gradient จาง)
- แกน X: เวลา (เปลี่ยนอัตโนมัติ: ชั่วโมง/วัน ตามช่วง)
- แกน Y: จำนวน event
- Tooltip: แสดงวันที่, ค่า
- สลับ: การเข้าชม | ผู้เข้าชม | เซสชัน (ปุ่ม pill ด้านบน)
- ฟิลเตอร์วันที่: วันนี้ / 7 วัน / 30 วัน / 90 วัน (พร้อม spinner loading)

### ตารางหน้ายอดนิยม
```
┌──────────────────────────────────────────────────────┐
│  หน้า              เข้าชม  ผู้เข้าชม  เข้าแล้วออก  เวลาเฉลี่ย  แนวโน้ม │
├──────────────────────────────────────────────────────┤
│  /                  4,821   3,211     32%      2m 17s   ─╱  │
│  /หวยรัฐบาลไทย       2,104   1,982     41%      0m 58s   ─╱  │
└──────────────────────────────────────────────────────┘
```
- คอลัมน์ตรงกัน (`table-layout: fixed` + `colgroup`)
- Sparkline 30 วัน ในคอลัมน์สุดท้าย
- เรียงลำดับได้ (คลิกหัวคอลัมน์)
- Pagination หน้าละ 20 รายการ
- URL ภาษาไทย decode แสดงผลถูกต้อง

### Realtime (ในหน้าภาพรวม)
```
┌─────────────────────────┬────────────────────────────┐
│  หน้าที่ใช้งานอยู่        │  เหตุการณ์ล่าสุด            │
│                         │                            │
│  /หวยรัฐบาลไทย     18   │  /หวยรัฐบาลไทย  ตอนนี้  เข้าชม │
│  /               15   │  /register    12 วิ ที่แล้ว  คลิก │
│  /หวยลาวสตาร์       9   │  /home       1 นาทีที่แล้ว  เลื่อน │
└─────────────────────────┴────────────────────────────┘
```
- Seed ข้อมูล 5 นาทีล่าสุดจาก DB ทันทีตอนเปิด
- อัพเดตผ่าน SSE ทุก 2 วินาที
- เวลาสัมพัทธ์อัพเดตทุก 10 วินาที

### กราฟช่องทาง (Funnel)
```
กดสมัคร       ████████████████████████  6  (67%)
              ↘ 33% หลุดออก
ขอ OTP       ████████████████         4  (75%)
              ↘ 25% หลุดออก
ยืนยัน OTP   ████████████             3  (67%)
              ↘ 33% หลุดออก
สมัครสำเร็จ   ████████                 2  (100%)
```
- แท่งแนวนอน สีไล่จากน้ำเงิน → ม่วง
- แสดง % หลุดออกระหว่างขั้นตอน (สีแดง)
- ฟิลเตอร์วันที่
- ใต้กราฟมี "โค้ดสำหรับ track ช่องทางนี้" (collapsible, มีปุ่มคัดลอกทุก step)

### การมีส่วนร่วม (แบ่งแท็บ)
- **ระดับการเลื่อน**: กราฟแท่งแนวนอน 4 สี (25%/50%/75%/100%) ต่อหน้า
- **การคลิก**: ตัวแปรคลิก (CRUD) + ตารางการคลิก (แสดงชื่อไทยถ้ามี)

### แหล่งที่มา (แบ่งแท็บ)
- **แหล่งทราฟฟิก**: Donut chart + ตาราง top referrer
- **อุปกรณ์**: ประเภทอุปกรณ์ + เบราว์เซอร์ + ระบบปฏิบัติการ

## รูปแบบการโต้ตอบ

### ฟิลเตอร์วันที่
- ค่าเริ่มต้น: 30 วันล่าสุด
- เลือกด่วน: วันนี้ / 7 วัน / 30 วัน / 90 วัน
- ปุ่มที่เลือกแสดง spinner ขณะโหลด
- ทุกกราฟอัพเดตพร้อมกัน
- ใช้ component `<DatePresets>` เดียวกันทุกหน้า

### Popup ยืนยัน
- ปุ่มสร้าง/ลบ ทุกจุด → แสดง popup ยืนยันก่อนเสมอ
- ปุ่มลบ: พื้นแดง, ข้อความ "ข้อมูลจะหายไปถาวร"
- ปุ่มสร้าง: พื้นน้ำเงิน
- กดนอก popup หรือปุ่มยกเลิก → ปิด popup

### การคัดลอกโค้ด
- ทุกจุดที่มีโค้ดตัวอย่าง → มีปุ่ม "คัดลอก"
- กดแล้ว → เปลี่ยนเป็น "คัดลอกแล้ว!" (สีเขียว, 2 วินาที)

### Realtime
- SSE connection ทำงานตลอดใน background
- ไฟเขียวกะพริบเมื่อ connected
- แสดง "กำลังเชื่อมต่อ…" ถ้าหลุด
- Auto-reconnect ด้วย exponential backoff

## สถานะว่าง (Empty State)

### ยังไม่มีข้อมูล
```
┌────────────────────────────────────┐
│         📡  (ไอคอน antenna)       │
│                                    │
│    รอรับเหตุการณ์…                 │
│                                    │
│    เพิ่ม script นี้ในเว็บของคุณ:    │
│    ┌──────────────────────────┐    │
│    │ <script src="...">       │    │
│    └──────────────────────────┘    │
│                                    │
│         [ คัดลอก ]                 │
└────────────────────────────────────┘
```

### ยังไม่มีเว็บไซต์
- Redirect ไปหน้า onboarding: ลงทะเบียนเว็บไซต์ → ได้ snippet → ตรวจสอบ event แรก

### ยังไม่มีช่องทาง
- แสดงข้อความ "ยังไม่มีช่องทาง" + ปุ่ม "สร้างช่องทางแรกของคุณ"

### ยังไม่มีตัวแปรคลิก
- แสดงข้อความ "ยังไม่มีตัวแปรคลิก กดสร้างเพื่อตั้งชื่อให้กับปุ่มที่ต้องการ track"

## Accessibility

- WCAG 2.1 AA ขั้นต่ำ
- กราฟทุกตัวมี data table เป็น fallback สำหรับ screen reader
- Keyboard navigation สำหรับทุก interactive element
- Focus indicator ชัดเจน contrast สูง
- สีไม่ใช่วิธีเดียวในการสื่อข้อมูล (ใช้ไอคอน/label ร่วมด้วยเสมอ)

## แนวทาง Animation

- ระยะเวลา: 150ms (micro), 300ms (transition), 500ms (กราฟเข้า)
- Easing: ease-out สำหรับเข้า, ease-in สำหรับออก
- เคารพ `prefers-reduced-motion` — ปิด animation ที่ไม่จำเป็น
- กราฟเส้นวาดเข้าตอนโหลดครั้งแรก (stroke-dashoffset)
- ตัวเลข KPI นับขึ้นตอนโหลด (requestAnimationFrame)
- Spinner หมุนบนปุ่ม date preset ขณะโหลดข้อมูล
